Convergence — GUID v4 → v7

Short summary: switching to v7 delivers gains immediately, no rebuild required.

.NET 11.0 Preview · SQLite B-Tree Proxy · 3 runs/point (median) · Legacy base: 100K v4 rows

The Problem

"My v4 base is fragmented. Do I need a rebuild before using v7?"

Test: Base with 100,000 v4 GUIDs (fragmented) + new inserts using Guid.CreateVersion7(), without modifying existing data.

Goal: find how many inserts the v7 Hybrid (one-line change) needs to match the v7 Full Rebuild (full migration).

Why does it converge so fast?

B-Tree separates new from old

Time ordering: v7 places a timestamp at the start, so each new v7 is greater than the previous.

B-Tree append-friendly: ordered inserts go to the tree tail. This creates new pages, avoiding conflicts with old ones.

Real impact: v4 noise affects old reads, not new v7 inserts. That is why gains appear immediately.

Practical Conclusion

One-line change already delivers the gain

Required change:

// Depois
